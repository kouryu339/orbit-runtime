以下是本轮可用工具。调用工具时，工具名和参数名必须严格使用下方列出的名称；具体输出格式以“响应格式（EXEC 行式协议）”为准。

上下文里可能出现由宿主发布的动态上下文章节，用于承载文档正文、页面结构、工作簿状态、PPTX 结构、业务对象状态等当前文本。该内容在 RPC 工具执行之外由宿主发布，通常比旧的工具返回或历史消息更接近当前事实。

当工具修改了这类状态时，工具结果可能先报告操作，而宿主随后才刷新动态上下文。应以之后发布的动态上下文为当前事实来源；RPC 工具不能直接读取或写入这些内容。

可以在工具调用前后附加少量用户可见说明。每个真实工具调用必须写成原始、独立的 EXEC 行：不要放进 Markdown 代码围栏，不要用反引号包裹，不要输出 Function Calling、tool_calls 或 JSON 参数。

**强制要求：每个参数名都必须带 -- 前缀。不要省略。**
正确格式示例：EXEC <下方列表中的工具名> --参数名 "参数值"
错误示例：ToolName param value；缺少 EXEC 和 -- 时运行时不会执行工具。

**重要：当参数值包含空格或特殊字符时，请用英文双引号包裹**，例如 --content "hello world" 或 --path "C:/Users/test"。
